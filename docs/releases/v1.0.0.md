# 🚀 Ubuntu Hello v1.0.0 Release

Welcome to the first official release of **Ubuntu Hello**! This version completes the rebranding from *Howdy* and brings full support for **Ubuntu 26.04 LTS**, a brand-new setup/onboarding wizard, a modernized GTK configuration interface, and seamless PAM/PolicyKit authentication integrations.

---

## ✨ Key Features in v1.0.0

*   **🔒 Windows Hello™-style Facial Authentication**: Use your built-in IR emitters and camera in combination with facial recognition to prove who you are for `sudo`, logins, lock screen, `su`, and graphical prompts.
*   **💻 Modern GTK Configuration GUI**:
    *   Interactive camera configuration & testing.
    *   Streamlined face profile manager (Add / List / Remove).
    *   New **Keyring Auto-Unlock** settings to automatically configure Face Auth for keyring unlocking.
    *   **Secure Password Verification**: Validates passwords securely via PAM before enabling keyring options.
*   **🧙 Onboarding Setup Wizard**: A user-friendly, multi-step graphical guide to walk you through configuring your camera, enrolling your face, and setting up keyring options upon installation.
*   **🔑 Secure Keyring Caching**:
    *   **TPM Hardware Sealing**: Seals keyring credentials directly in TPM security hardware if available.
    *   **Software Fallback**: Uses machine-specific `machine-id` encryption for secure offline credential storage when a TPM is absent.
    *   **Deferred Caching**: Leverages a PAM module hook (`pam_sm_setcred`) to capture/update keyring credentials automatically during subsequent login/authentication events.
*   **⚡ Automated Installer**: Single-command shell script (`install.sh`) that automates dependency builds, AI model downloads (Dlib models), PAM configuration, and Polkit integration.
*   **⚙️ System Integration**: Fully compliant PAM module and custom Polkit policy files for secure elevated execution in Ubuntu App Center and administrative tasks.

## 📥 Installation

### Method 1: Quick Install (Recommended)
Open your terminal and run the universal installer:
```bash
curl -fsSL https://raw.githubusercontent.com/ventura8/ubuntu-hello/master/install.sh | sudo bash
```

### Method 2: PPA (Ubuntu / Linux Mint)
```bash
sudo add-apt-repository ppa:ventura8/ubuntu-hello
sudo apt update
sudo apt install ubuntu-hello
```

### Method 3: Debian / Manual Packaget
Download the attached `.deb` package for your architecture, then install it using `apt` or `gdebi`:
```bash
sudo apt install ./ubuntu-hello_1.0.0-1ppa7_amd64.deb
```

## 🛠️ Getting Started

1.  **Enroll your face**:
    ```bash
    sudo ubuntu-hello add
    ```
2.  **Test the authentication**:
    Open a new terminal session and run:
    ```bash
    sudo -i
    ```
    *(The camera should light up and authenticate you instantly!)*

## 📋 Changelog (v1.0.0)

*   **Rebranding**: Renamed codebase, assets, commands, and packages from *Howdy* to **Ubuntu Hello**.
*   **Platform Support**: Added compatibility and fixed compilation issues on **Ubuntu 26.04 LTS (Plucky / Resolute)**.
*   **Keyring Auto-Unlock**: Fully implemented automated GDM keyring unlocking via face authentication, with full support for sealing credentials using TPM hardware or machine-id encrypted software fallback.
*   **PAM Password Verification**: Integrated secure system-level password verification for keyring setups using `/etc/pam.d/ubuntu-hello-verify`.
*   **Deferred Caching**: Implemented a deferred password caching mechanism using PAM's `setcred` hook to automatically seal credentials on subsequent logins.
*   **CLI Subcommands**: Added `keyring` and `set` commands.
*   **Bug Fixes**:
    *   Resolved PAM/PolicyKit authentication hangs.
    *   Fixed build warnings in the Meson build pipeline.
    *   Fixed automated GPG signing in the PPA release workflow.
    *   Fixed deprecated datetime calls (`utcnow()`).
