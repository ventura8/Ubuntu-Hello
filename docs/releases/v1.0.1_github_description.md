# Ubuntu Hello v1.0.1 - Security Hardening & Maintenance Release

This release brings critical security hardening, bug fixes, and filesystem permission locks to **Ubuntu Hello** to ensure secure operation within PAM, system administration, and TPM integrations.

---

## 🔒 Security Hardening

*   **Strict Input Validation**:
    *   Added robust regex-based validation (`^[a-zA-Z0-9_.][a-zA-Z0-9_.-]*\$?$`) to all username inputs across the C++ PAM module and the Python CLI, blocking any potential command injection or argument manipulation vectors.
*   **TPM Context Relocation**:
    *   Relocated all temporary TPM context files (`primary.ctx`, `p_<pid>.ctx`, `s_<pid>.ctx`) from the globally-writable `/tmp` folder to a root-only directory `/etc/ubuntu-hello/tpm-keys/` managed under `0700` permission controls, preventing local symlink hijacking and directory harvesting attacks.
*   **Safe Subprocess Execution**:
    *   Completely eliminated all `shell=True` and `subprocess.getstatusoutput` invocations in the administration GTK GUI helper scripts, replacing them with structured list-based executions to mitigate command injection risks.
*   **Secure Staging Builds**:
    *   Hardened the universal `install.sh` installation script by switching build directories to secure, randomized staging paths created using `mktemp -d`.

## 📁 Access Controls & File Permissions

*   Configured models (`/etc/ubuntu-hello/models/`) and keyring keys (`/etc/ubuntu-hello/tpm-keys/`) storage folders with strict `0700` (root-only) permissions.
*   Updated face database `.dat` profile registration to enforce `0600` permissions upon file creation.

## 🛠️ Bug Fixes

*   **Liveness Tracking**: Fixed a mathematical average formula bug in initial eye-distance calculations for nod-based liveness verification.
*   **Codebase Cleanup**: Purged deprecated packaging folders inside subprojects (`ubuntu-hello/debian/` and `ubuntu-hello-gtk/debian/`) to streamline the source tree.
